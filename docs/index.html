Introduction

L'API Real Estate agrège les annonces immobilières de multiples sources en France (portails, réseaux d'agences, agences indépendantes) et les expose via une API REST unifiée.

Chaque annonce est dédupliquée — un même bien vu sur plusieurs sites est regroupé en une seule entrée (canonical ad) avec la liste de ses sources.

Swagger UI — Une documentation interactive complète est disponible à {BASE_URL}/docs.

Authentification

Toutes les routes /api/* nécessitent un header X-API-Token.

curl -H "X-API-Token: VOTRE_TOKEN" https://api.example.com/api/ads

Le token est fourni lors de l'onboarding. Chaque token est lié à un compte avec un quota de requêtes.

Ne partagez jamais votre token. Il est personnel et donne accès à l'ensemble des données de votre plan.

Base URL

https://api.example.com

En développement local : http://localhost:8000

Limites de requetes

Les limites dépendent de votre plan :

PlanRequetes / minRequetes / jourTiers accessibles
Gratuit10500Tier 2
Standard605 000Tier 2 + 3
Premium12050 000Tier 2 + 3 + 4
Tier 1 (LeBonCoin, SeLoger, BienIci, Zefir) n'est jamais accessible via l'API. Ces données sont à usage personnel uniquement pour des raisons juridiques.

En cas de dépassement, l'API retourne 429 Too Many Requests avec un header Retry-After.

Rechercher des annonces

GET /api/ads
Recherche paginée dans les annonces immobilières avec filtres multiples.

Paramètres

ParamètreTypeDescription
citystringNom de ville (insensible à la casse, converti en majuscules). Ex: TOURS
zipcodestringCode postal ou préfixe. 37 = tout le département, 37000 = ville exacte
price_minintPrix minimum en euros
price_maxintPrix maximum en euros
surface_minfloatSurface minimum en m²
surface_maxfloatSurface maximum en m²
rooms_minintNombre minimum de pièces
rooms_maxintNombre maximum de pièces
property_typestringappartement, maison, terrain, parking
prixm2_maxintPrix max au m² (annonces sans prix/surface sont conservées)
tierint[]Filtrer par tier (1-4). Répétable : ?tier=2&tier=3
qstringRecherche textuelle dans le titre
pageintPage (défaut: 1)
per_pageintRésultats par page (défaut: 50, max: 200)

GET /api/ads?zipcode=37&property_type=appartement&price_max=120000&surface_min=30&per_page=20

Systeme de tiers

Les sources sont organisées en 4 niveaux. Votre plan détermine les tiers accessibles.

TierSourcesAccès
Tier 1LeBonCoin, SeLoger, BienIci, ZefirUsage personnel uniquement — jamais via API
Tier 2PAP, entreparticuliers, open dataTous les plans
Tier 3Century21, IAD, Guy Hoquet, ERA, Square Habitat, Human ImmobilierPlan Standard+
Tier 4Agences indépendantes/régionalesPlan Premium

Utilisez le paramètre tier pour filtrer : ?tier=2&tier=3 retourne uniquement les annonces de ces tiers.

Exemples de code

cURL

curl -s \
  -H "X-API-Token: VOTRE_TOKEN" \
  "https://api.example.com/api/ads?zipcode=37&property_type=appartement&price_max=120000&per_page=10" \
  | python3 -m json.tool

Python

import requests

API_URL = "https://api.example.com"
TOKEN = "VOTRE_TOKEN"

response = requests.get(
    f"{API_URL}/api/ads",
    headers={"X-API-Token": TOKEN},
    params={
        "zipcode": "37",
        "property_type": "appartement",
        "price_max": 120000,
        "surface_min": 30,
        "per_page": 20,
    },
)

data = response.json()
print(f"{data['total']} annonces trouvees")

for ad in data["items"]:
    print(f"  {ad['title']} — {ad['price']}€ — {', '.join(ad['source_names'])}")

JavaScript (fetch)

Pagination

Toutes les listes sont paginées. La réponse contient :

  • total — nombre total de résultats
  • page — page courante
  • per_page — résultats par page

Pour parcourir les résultats :

# Page 1
GET /api/ads?page=1&per_page=50

# Page 2
GET /api/ads?page=2&per_page=50

# Dernière page
total_pages = ceil(total / per_page)
Maximum 200 résultats par page. Au-delà, utilisez la pagination pour parcourir les résultats.
